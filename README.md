# learnings

Documenting my learnings throughout my software engineering career — design patterns,
system architecture, databases, and development skills & best practices.

All content is written in Markdown. See [STYLE.md](STYLE.md) for conventions on how
notes are named, formatted, and added.

## Contents

- [Design Patterns](design-patterns/) — creational, structural, behavioral, and beyond.
- [System Design](system-design/) — architecture styles, scalability, distributed systems.
- [Databases](databases/) — data modeling, indexing, transactions, SQL & NoSQL.
- [Best Practices](best-practices/) — code quality, testing, git, reviews, dev workflow.
- [Languages](languages/) — language- and runtime-specific learnings.

## How to use this repo

Each topic folder has its own `README.md` acting as a section index. Individual
learnings live as `kebab-case.md` files inside the relevant folder. When adding a
note, link it from the section index so everything stays discoverable.
